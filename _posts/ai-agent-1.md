# AI智能体学习笔记 (一)：基本概念

## 1. 提示词 (Prompt)

**定义**：你发送给 AI 的每一句话，无论是问题、指令还是背景信息，都是 Prompt。

| 类型                           | 作用                           | 类比                                           |
| :----------------------------- | :----------------------------- | :--------------------------------------------- |
| **用户提示词** (User Prompt)   | 直接的问题或请求               | “我肚子疼怎么办？”                             |
| **系统提示词** (System Prompt) | 设定 AI 的角色、风格和能力边界 | “您现在是一位温柔的医疗助手，请用通俗语言解答” |

## 4. 智能体 (AI Agent)：会“动手”的智能助手

    AI Agent 不仅仅是一个聊天框，它是一个能利用工具自主完成复杂任务的程序。
    
    Agents combine language models with tools to create systems that can reason about tasks, decide which tools to use, and iteratively work towards solutions.
### Agent 三要素

| 要素     | 功能                                     |
| -------- | ---------------------------------------- |
| 大脑     | 大语言模型 (LLM)负责推理和规划           |
| 工具     | 可调用的外部工具(搜索、计算、读写文件等) |
| 工作流程 | ReAct 循环(推理→行动→观察→循环)          |

示例：你让 Agent “帮我找某应用的安装目录”。它会先列出文件、读取目录，反复尝试直到找到为止。



## 5. 模型上下文协议 (MCP)：AI 界的“USB 接口”

    定义：MCP (Model Context Protocol) 是一种通用的通信标准，标准化了智能体与工具服务之间的通信，让不同的 AI Agent 能使用同一套工具库。
    
    - MCP Server (服务端)：提供工具或数据的仓库。
    - MCP Client (客户端)：调用工具的智能体。
    - 示例：一个通用的“web_browse”工具可以被部署为 MCP Server，所有的 AI 助手只要接入 MCP 就能学会上网。

```
┌─────────────────────────────────────────────┐
│            AI Agent (MCP Client)            │
│    “我需要搜索网页，调用 web_browse 工具”    │
└───────────────┬─────────────────────────────┘
                │ 通过 MCP 协议请求
                ▼
┌─────────────────────────────────────────────┐
│        工具服务端 (MCP Server)              │
│   ┌─────────┐ ┌─────────┐ ┌─────────┐      │
│   │网页搜索 │ │文件操作 │ │计算器   │      │
│   │web_browse││file_ops │ │calculator│      │
│   └─────────┘ └─────────┘ └─────────┘      │
└─────────────────────────────────────────────┘
```



### 通信方式 
- MCP 协议支持两种主要的物理连接方式：
  - 标准输入/输出 (Stdio)：当 MCP Server 与 Agent 运行在同一台机器上时使用。
  - 网络通信 (HTTP)：当 MCP Server 部署在远程服务器或云端时使用。
  



 ### 交互流程 

  1. 工具检索：Agent (MCP Client) 遵循 MCP 协议从 MCP Server 获取所有可用工具的描述信息。
  2. 协议转换：Agent 将这些工具信息转化为 System Prompt 或 Function Calling 格式发送给 AI 模型。
  3. 决策生成：AI 模型根据用户问题，决定调用特定的工具（如 web_browse）。
  4. 指令传递：Agent 接收到模型的调用请求，通过 MCP 协议向 MCP Server 发送执行指令。
  5. 结果反馈：MCP Server 执行任务并返回数据，由 Agent 转发给 AI 模型进行最终汇总。
